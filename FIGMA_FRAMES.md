## Recommended Frame Size

- Desktop: `1440 x 1200`
- Mobile bonus: `390 x 844`

## Spacing System

- Base grid: `8px`
- Common spacing: `8 / 16 / 24 / 32 / 48`
- Card radius: `16px` to `24px`

## Type System

- Font: `Inter`
- H1: `32 / 40`, weight `700`
- H2: `24 / 32`, weight `600`
- H3: `20 / 28`, weight `600`
- Body: `16 / 24`, weight `400`
- Small: `14 / 20`, weight `500`

## Color System

- Primary: `#1D4ED8`
- Secondary: `#2563EB`
- Background: `#F8FAFC`
- Card: `#FFFFFF`
- Border: `#E5E7EB`

- Error: `#EF4444`

## Page 01: UX Research

### Frame: `Problems`

Add four text blocks:

- Long confusing forms
- No guidance for first-time users
- Document requirements are unclear
- Appointment booking feels disconnected

### Frame: `Pain Points`

Add quote cards:

- "What documents do I need?"
- "Did my form get saved?"
- "How long will this take?"
- "Where do I book my slot?"

## UX Notes to Add in Figma

Add a final frame called `Design Decisions` with these bullets:

- Reduced cognitive load using a step-by-step flow
- Split identity and address into separate stages
- Added autosave feedback to reduce fear of lost progress
- Introduced a document checklist before upload to remove ambiguity
- Moved booking after review so the sequence feels more natural
- Added confirmation, receipt access, and dashboard record visibility for trust
